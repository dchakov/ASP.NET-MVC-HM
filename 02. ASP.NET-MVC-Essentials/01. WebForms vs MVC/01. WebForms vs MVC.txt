similarities:
They both share API's related to http and user authentication.
Have the same startup events in Global.asax.
Both can be developed through the same IDE 
Use the same NuGet packages for development.


differences:
Web forms is component-based and relies on controls, postback(event driven) and has less control over HTML.
Web forms is hard to test.
MVC is testable loosely coupled and extensible.
MVC provides more control over the HTML output and it looks a lot cleaner.
They use different View Engines.
MVC has better separation of concerns.
MVC has clean URLs (REST-Like)
